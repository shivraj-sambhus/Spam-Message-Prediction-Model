# 1. Objective

I want to build a predictive model that classifies a text message (SMS) as spam or not spam. In this project, I will use an unstructured text dataset from 2012 found on the UC Irvine machine learning repository that contains SMS data from students of the National University of Singapore. Since the text is unstructured, I will need to first extract meaningful features from it and create a structured version of it. I will do this by creating an ETL pipeline and loading it in PostgreSQL to generate a materialized view used in the predictive model. To select the final model, I will use an iterative approach involving logistic regression, decision trees, and an XGBoost model.

Link to the data: https://archive.ics.uci.edu/dataset/228/sms+spam+collection

# 2. ETL and Feature Selection

Let's first take a look at the data. All we can really see is each message and its corresponding label. The messages seem full of typos and punctuation errors, typical of spam data. `ham' refers to non-spam messages.

```
ham	Go until jurong point, crazy.. Available only in bugis n great world la e buffet... Cine there got amore wat...
ham	Ok lar... Joking wif u oni...
spam	Free entry in 2 a wkly comp to win FA Cup final tkts 21st May 2005. Text FA to 87121 to receive entry question(std txt rate)T&C's apply 08452810075over18's
ham	U dun say so early hor... U c already then say...
```

First, I'll extract the data using Pandas, and create two columns: label and message. There are 5572 rows and 2 columns. There are significantly more ham messages (87.3%) than spam messages (12.6%).

Next, I'll transform the data by cleaning it and generating features to be used in the predictive model. For data cleaning, I'll drop empty rows and duplicates and also remove whitespace and trailing characters. This will be useful when creating features in the dataset like character count and word count. For feature creation, I'll use simple FOR loops to iterate through each message and create a numeric label (1/0) for spam/ham, character count, word count, spam word count, and a label to indicate the presence of hyperlinks in the message. The dataset is now structured.

Next, I'll load the data to PostgreSQL by using the sqlalchemy library.

# 4. Iterative Approach

The SMS data is now cleaned, structured, and ready for analysis. I will now use the scikit-learn library in Python to compare three different predictive models to classify each SMS as spam or ham: logistic regresion, decision tree, and XGBoost.

I used TF-IDF text vectorization to measure the relevance of each word in the text messages, and included this in the testing and training data. This will help the models classify each message as it can know how relevant certain words are in the spam or ham categories. I assigned 20% of the data for testing, and 80% of the data for testing.

# 5. Definitions

Before I jump into the model comparison, let's define some of the basic terms in the output.

ROC-AUC: The receiver operating characteristic and area under the curve. It represents the rate of distinguishing between a spam and ham message, given the proportion of each in the dataset. It is more reflective of the data as compared to accuracy, since the dataset contains more ham messages than spam.

Accuracy: The rate of distinguishing between a spam and ham message. Remember that this does not account for the proportion of spam and ham messages.

Precision: The rate of predicting the spam/ham messages that are actually spam/ham in the entire dataset.

Recall: The rate of predicting spam/ham messages that were identified in the entire dataset.

F1 Score: The rate of balancing between precision and recall.

Support: The number of spam/ham messages in the testing dataset. Remember that we assigned 80% of the data for testing.

Macro Average: The score given to each of the above categories in terms of equally weighted data points. I will use this instead of the weighted average so that we don't ignore the spam data points. 

Weighted Average: The score given to each of the above categories in terms of their proportion in the dataset. I won't use this as there are significantly more ham messages than spam messages and so this score will poorly reflect the spam messages.

# 6. Insights

The XGBoost model has the highest ROC-AUC score, precision, recall, F1 score, and accuracy as compared to the other models. It can clearly distinguish between the spam and ham messages and can understand that there are more ham messages than spam. This is because of XGBoost's iterative approach which allows it to reduce its error as it classifies each message. Note that the recall rate for ham messages is approximately the same for all three models because they form the majority of the dataset. Because of this, I don't think that the accuracy of each model is important as compared to other outputs such as the ROC-AUC score. 

In conclusion, the XGBoost model outperforms the logistic regression and decision tree models in terms of classifying each SMS as ham or spam because its ROC-AUC score and classifcation report statistics are the best.
